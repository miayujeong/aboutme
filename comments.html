COMMENTS

Reflection on overall design

Overall, I made an effort to ensure that every page on this website provides a sense of unity for users. I used similar structure including the same top navigation bar across all pages and limited the color palette, including the background color. Additionally, elements have been added to assist diabled user exploring the website.

Technical aspects on my website

Structure

All pages of this website has a common structure: a top navigation bar, a header (except for the index page), contents, and a footer. These individual elements are styled separately using <div> tags.

Top naviation bars

Pages of this website can be navigated through the navigation bar. Each item uses the <a> tag to possess the link to other page. They are set style for the hover state using :hover property.

Parallel columns

Exploring W3cschool website, I learned how to make webpage with equal or unequal columns. This structure is applied to the INDEX and the FUTURE page. On both page, there is a parent element that wraps columns which has flex display property. And each child element, erving as individual columns, has its width set. Then, clear any whitespace with :after property. 

.future-img {
            display: flex;
                      
            .col.left {
              width: 45%;
            }
          
            .col.right {
              width: 55%;
            }

            .future-img:after {
              content: "";
              display: table;
              clear: both;
            }
          }

CSS selectors: class

I used class for objects I might reuse again. One instance is the images on the PAST page. I used class="small-img" for every image here, to apply same style (round border and size) more effectively.

Aesthetics of my website

Color scheme

I primarily used navy blue, light sky blue, and light gray across all pages. This color scheme was chosen to create a professional impression of an IT industry.

Layout

At the very top of each page is a navigatio bar, allowing users to easily navigate to their desired page. Except for the index page, every page has a header section at the top, making it easy for users to notice which page they are currently viewing.

On the PAST page, I used a table to organize my experiences in a consistent format. On the left side, there is a simple image related to the activity, and on the right side, details are listed in a uniform order and style. This structure allows users to quickly understand the content without additional explanations.

Consistency

As mentioned above, this website provides a sense of consistency through a common layout and color palette across all pages. Each page has the same navigation bar in the same position, with the contents below and a footer centered at the very bottom. Additionally, the title and an icon appears next to it maintain the same format, also contributing to the uniformity of the website overall.

Readability

By leaving sufficient white spaces between elements on each page, I ensured the readability of the website. The space between a paragraph and next heading is larger than the space between regular paragraphs, to indicate the difference in semantic. The INDEX page has more whitespace than COMMENT page, which is because they are distinct in contents. While the COMMENT page contains a lot of texts and technical information, the INDEX page is lighter and intended to be the first landing page for users.Therefore, I added more space on the INDEX page to create a less overwhelming and more comfortable feeling for users.

Accessibility aspects of my website

ALT

All images on this website inculde alternative text to help disabled users who access the website only through screen readers. For example, the mimoji profile image on the INDEX page contains the following alternative text: 

<img src="..." alt="profile image" />

Color contrast

I avoided using low-contrasted colors for users who have low vision. I checked the color contrast of main colors I used in this website through WebAIM Contrast Checker , to make sure that they have enough contrast ratio.

Structure

All contents in this website fit into the logical structure by using heading tags for heading texts. Also, they follow a meaningful order, which aids screen readers in navigating the website more smoothly.

Also, the navigation bar is created using <ul> and <li> tags to indicate its semantic as the list of links to screen readers.

Links

The hyperlinks in this website are recognizable compared to plain text by different color. I used link class to apply distinct color style (blue) to texts that serves as a hyperlink. 

.link {
          color: #006a95;
          text-decoration: none;
        }

Paragraphs

By limiting the max width of paragraph or the table, it is made sure that the line lengths of paragraphs in this website are not too long.
